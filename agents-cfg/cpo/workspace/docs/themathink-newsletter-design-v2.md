## 二、目标用户画像

### 2.1 核心用户

| 画像 | 特征 |
|------|------|
| **AI 创业者** | 30-40岁，正在构建 AI 产品，需要行业洞察 |
| **技术从业者** | 25-35岁，工程师/产品经理，想了解 AI 趋势 |
| **独立思考者** | 28-45岁，对 AI 感兴趣，喜欢深度阅读 |
| **投资人** | 30-50岁，寻找 AI 领域的投资机会 |

### 2.2 用户痛点细化

| 痛点 | 场景 | 解决方案 |
|------|------|----------|
| 信息焦虑 | 每天刷 AI 新闻，越看越慌 | 精选 + 深度分析，去除噪音 |
| 只知其一 | 知道 AI 热点，但不懂背后逻辑 | 多维度解读，哲学 + 商业 + 技术 |
| 无法行动 | 看了很多但不知道该做什么 | 提供行动建议 |

## 三、内容体系

### 3.1 内容分类矩阵

```
                    深度浅        
               ◀─────────────▶
                    
  短        │   快讯    │   解读    │   专题    │
  ──────────┼───────────┼───────────┼──────────┤
  快        │  Signal   │  Insight  │  Deep    │
            │  (信号)   │  (洞察)   │  (深度)  │
            
  长        │   周报    │   专栏    │   课程    │
```

### 3.2 栏目详细定义

#### 📡 Signal（信号）- 5分钟

```
目的：让用户快速了解本周发生了什么

内容：
├── AI 大事件（1条）
│   • 本周最重要的 AI 新闻
│   • 50字内事件描述
│   • 1句话点评
│
├── 值得注意的小变化（3-5条）
│   • 新产品 / 新功能
│   • 重要的论文
│   • 行业趋势
│
└── 本周数据
    • AI 领域融资 / 收购
```

#### 💡 Insight（洞察）- 5分钟

```
目的：提供深度分析和独特视角

内容：
├── 封面主题（本期核心）
│   • 一个核心话题
│   • 800-1000字
│   • 多维分析（技术 + 商业 + 哲学）
│   • 独特的观点
│
└── 读者问答（可选）
    • 精选读者问题
    • 深度回答
```

#### 🛠️ Toolkit（工具箱）- 2分钟

```
目的：提供实用资源

内容：
├── 新工具推荐（1-2个）
│   • 工具名称
│   • 用途
│   • 推荐理由
│
└── 资源下载（可选）
    • 行业报告
    • 精选文章
```

## 四、内容风格指南

### 4.1 品牌声音

| 维度 | 定义 |
|------|------|
| **语调** | 专业但不高冷，深刻但不晦涩 |
| **人称** | "你"，偶尔用"我们" |
| **风格** | 思考者视角，像在和聪明朋友聊天 |
| **长度** | 句子短，段落短，允许留白 |

### 4.2 文风示例

**❌ 不要这样写：**
> 近日，OpenAI 发布了 GPT-5 模型，该模型在多项基准测试中取得了 SOTA 成绩，展现了强大的推理能力和多模态理解能力...

**✅ 要这样写：**
> 本周，OpenAI 发布了 GPT-5。
> 
> 重点不是它又变强了，而是它开始"慢思考"了。
> 
> 这意味着什么？

### 4.3 标题风格

```
格式：emoji + 核心观点 +（可选补充）

示例：
• GPT-5 的真正突破，不在技术
• AI 创业的窗口，可能只剩 6 个月
• 为什么我不担心 AI 会取代人类
• 2026 年 AI 领域的 3 个确定性趋势
```

## 十、风险与预案

| 风险 | 影响 | 预案 |
|------|------|------|
| 打开率低 | 用户不感兴趣 | 优化标题 + 缩短内容 |
| 退订率高 | 内容不匹配 | 增加退订原因收集 |
| 发送失败 | 技术问题 | 备用发送渠道 |
